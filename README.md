## socketIO

```
웹 소켓이란?
- HTML5 에 새로 추가된 스펙으로 실시간 양방향 데이터 전송을 위한 기술
- HTTP와 다르게 ws(web socket)이라는 프로토콜을 사용

웹 소켓이 나오기 이전의 실시간 데이터 전송 방식
1. 폴링(polling) 방식
- HTTP가 클라이언트에서 서버로 향하는 단방향임
- 주기적으로 서버에 새로운 업데이트가 있는지 확인 요청을 보내는 있으면 가져오는 단순 무식한 방법

2. SSE (Server Sent Event)
- EventSource 라는 객체 사용
- 클라이언트에서 서버로 데이터를 보낼 수 없음
- 서버에서 클라이언트로 데이터를 보내는 단방향 통신

3. web socket
- 양방향 통신
- 많은 데이터들을 주고 받을 때 사용하기 좋다.

4. SocketIO
- 양방향 통신
- web socket을 편리하게 사용할 수 있도록 도와주는 라이브러리
- 많은 유저와 방들을 다루기에 좋다. (편의 기능 제공)
```
